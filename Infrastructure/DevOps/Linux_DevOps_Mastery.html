The Linux Universe

Understanding the foundation of modern server infrastructure from scratch.

1. What is an Operating System (OS)?

Think of an OS as the "Manager" of a company. It coordinates everything between the hardware (the office building and machines) and the software (the employees performing tasks). Without it, the computer is just a pile of metal and silicon.

Examples: Windows (the most common for home/office), macOS (for creators), and Linux (the backbone of the internet).

2. What is the Linux Kernel?

The Kernel is the heart of the operating system. It's the silent worker that manages the CPU, Memory, and Devices. When you save a file, the Kernel is the one actually talking to the hard drive.

Kernel vs. User Space: To keep the system safe, Linux separates the "VIP Room" (Kernel Space, where hardware is touched) from the "Public Area" (User Space, where your apps like Chrome or VS Code live). Apps must ask the Kernel for permission via "System Calls" to do anything important.

2b. Deep Dive: The Linux Boot Journey

What happens when you press the Power Button? It's a precisely choreographed 4-stage journey:

  • BIOS/UEFI: The hardware wakes up and runs a Power-On Self-Test (POST). It then looks for a bootable device.
  • GRUB (Bootloader): This menu lets you choose which OS to load. Its main job is to find the Kernel on your disk and load it into RAM.
  • Kernel Initialization: The Kernel takes control, probes the hardware, and mounts the "Root File System" (/).
  • Init/Systemd: The very first process (PID 1) starts. It wakes up all the other "Daemons" (background services) like your network manager or desktop environment.

3. How to check Kernel version?

Use the command uname -a. This prints system information including the Kernel version, release, and machine architecture.
$ uname -a Linux devops-srv 6.5.0-18-generic #18-Ubuntu SMP ... x86_64

Navigating the File System

Mastering the art of moving around and managing files like a pro.

4. Navigation Trio: cd, pwd, and ls

These are your legs in the Linux world. Use cd to move (Change Directory), pwd to see where you are (Print Working Directory), and ls to look around (List).

Think of it like: ls is opening your eyes, pwd is looking at most a map, and cd is walking to another room.

5. Creating Files & Folders

Use touch filename to create an empty file (or update the timestamp if it already exists). Use mkdir dirname to create a fresh directory.

$ touch script.sh $ mkdir backup_logs

6. Copy, Move, and Rename

cp copies, mv moves (or renames). If you "move" a file to a new name in the same place, you've effectively renamed it!

Rename Example: mv old_name.txt new_name.txt

7. Deleting Safely

rm deletes files. rmdir deletes empty directories. To delete a folder with everything inside it, use rm -rf foldername (Be extremely careful!).

Caution: Linux doesn't have a "Recycle Bin". Once you rm, it's gone.

7b. Deep Dive: What is an Inode?

To Linux, a file's "name" is just a label. The real "ID card" of a file is the Inode. It stores everything EXCEPT the filename and the actual data:

  • File Type and Permissions
  • Owner and Group IDs
  • File Size and Timestamps (Created, Modified, Accessed)
  • Pointers to where the data blocks are on the disk.
Why it matters: If you run out of Inodes, you can't create new files, even if you have 100GB of free disk space!

7c. The FHS (Filesystem Hierarchy Standard)

Linux follows a strict map. Every folder has a purpose:

  • /etc: Configuration files (The system's brain).
  • /bin: Essential binaries (ls, cp, rm).
  • /var/log: Where every event is recorded.
  • /home: Where individual users store their personal files.
  • /root: The home for the superuser.

Users, Permissions & Security

Who belongs here? What can they do? Let's find out.

8. Who is 'root'?

The Root user is the "Superuser" — the absolute god of the system. Root can delete anything, install anything, and change any setting. It's dangerous to stay as root all the time!

Standard Prompt: $ for regular users, # for root.

9. The sudo Command

Short for "SuperUser DO". It allows a regular user to perform one task with root privileges. It's like having a temporary key to the VIP room.

$ sudo apt update

10. Permissions: chmod and chown

chmod (Change Mode) changes what can be done (read, write, execute). chown (Change Owner) changes who owns the file.

The Permissions Code: Read (4), Write (2), Execute (1). Add them up: 7 (rwx), 5 (r-x).

11. File: /etc/passwd

This isn't where passwords are! It's actually a database of all user accounts on the system, their IDs, and their default shells.

Real Passwords: Are stored securely in /etc/shadow.

11b. Advanced: SUID, SGID, and Sticky Bit

Beyond the standard rwx, Linux has special permissions for advanced scenarios:

  • SUID (Set User ID): When run, the program "borrows" the permissions of the file owner (often root). Example: passwd needs to write to /etc/shadow.
  • SGID (Set Group ID): Similar to SUID but for the group. In a directory, new files inherit the directory's group.
  • Sticky Bit: Used on shared folders like /tmp. It ensures only the owner of a file can delete it, even if others have write access to the folder.

11c. Professional SSH Hardening

Securing your front door is critical. Always do these 3 things in /etc/ssh/sshd_config:

  1. PermitRootLogin no (Never let root log in directly).
  2. PasswordAuthentication no (Use SSH Keys only).
  3. Port 2222 (Move away from default port 22 to avoid bots).

Advanced Command Line Tools

The "Power Tools" of Linux. Master these to automate almost anything.

12. grep: The Search Master

Grep stands for "Global Regular Expression Print". Use it to find text inside files. It's like 'Ctrl+F' but for your entire system.

$ grep "error" /var/log/syslog
Useful Flag: grep -i makes the search case-insensitive.

13. sed & awk: The Content Shapers

sed is a stream editor (great for find-and-replace). awk is a full programming language for processing text in columns/tables.

awk Example: awk '{print $1}' file.txt prints only the first column.

14. Pipes (|) and Redirection (>)

A Pipe takes the output of one command and feeds it as input to another. Redirection (>) saves the output to a file.

Combined: ls -l | grep ".sh" > shell_files.txt

15. find: The Detective

Use find to search for files based on names, size, permissions, or time. It's incredibly deep and powerful.

$ find /home -name "*.jpg"

Networking & Infrastructure

Connecting the dots. How servers talk and how to fix them when they don't.

16. What is an IP vs. Gateway?

An IP Address is your server's unique home address. The Gateway is the "front door" of your network that leads to the rest of the internet.

Analogy: Your house has a distinct address (IP), but you always leave through the main gate (Gateway) to reach the city.

17. DNS: The Internet's Phonebook

DNS (Domain Name System) translates human names like google.com into IP addresses like 142.250.190.46.

Troubleshooting: Use dig or nslookup to check DNS records.

18. ping & traceroute

ping checks if a server is alive. traceroute shows you every "stop" (router) a packet makes on its way to the destination.

If ping fails: The server might be down, or a firewall might be blocking your "knock".

19. netstat & ss

These tools show you what "ports" are open on your server. It's like checking which windows and doors are unlocked in your house.

$ ss -tuln

19b. The OSI Model: The 7 Layers of Networking

How data travels across the world, visualized in 7 steps:

7. ApplicationHTTP, SSH, DNS (What you interact with)
6. PresentationEncryption, Compression (SSL/TLS)
5. SessionStart/Stop connections
4. TransportTCP (Reliable) vs UDP (Fast)
3. NetworkIP Routing (The logic)
2. Data LinkMAC Addresses, Switches (The wiring)
1. PhysicalCables, Fiber, Wi-Fi radio waves

19c. The TCP 3-Way Handshake

Before a server and client talk, they must "agree" to talk:

  1. SYN: Client says "Hey, can we talk?"
  2. SYN-ACK: Server says "Yes, I heard you. Can YOU hear me?"
  3. ACK: Client says "Yes, loud and clear. Let's start!"

System Management & Monitoring

Keeping the lights on and the engine running smoothly.

20. Processes: ps, top, and kill

Everything running on Linux is a Process. ps lists them, top shows them in real-time, and kill stops them.

Zombie Processes: These are "ghosts" in your system. They've finished their work but are still hanging around in the process table waiting for their parent to notice them.

21. Log Management: /var/log

Linux is a big talker; it logs everything. Most logs live in /var/log. Use tail -f to watch a log file live as it updates.

$ tail -f /var/log/nginx/access.log

22. Storage: df and du

df (Disk Free) shows how much space is left on your "hard drives". du (Disk Usage) shows how much space a specific folder is taking up.

Beginner Trick: df -h makes the numbers "Human Readable" (GB instead of bytes).

23. Scheduling: crontab

Need to run a script every night at 2 AM? Use crontab. It's the "alarm clock" of the Linux world.

Syntax: Minute Hour Day Month Day-of-Week (* * * * *)

Modern DevOps Mastery

Going beyond the basics into the world of automation and scale.

24. Containers: What is Docker?

Containers are like "lightweight hotels" for your app. They package everything the app needs to run, so it works exactly the same on your laptop as it does in the cloud.

Analogy: If a Virtual Machine is a whole house with its own pipes and wires, a Container is an apartment in a building sharing the core plumbing (the Linux Kernel).

25. CI/CD: The Assembly Line

Continuous Integration (CI) and Continuous Deployment (CD) automate the process of testing and shipping code. It's like a factory assembly line for software.

Tools: GitHub Actions, GitLab CI, and Jenkins are the big players here.

26. Orchestration: Kubernetes (K8s)

If Docker provides the containers, Kubernetes is the "Captain" of the ship that decides where those containers should go, how they should talk, and what to do if one crashes.

  • Pods: The smallest unit in K8s. Think of it as a wrapper around one or more containers.
  • Deployments: Tell K8s how many copies (replicas) of your app should be running. If one dies, K8s automatically starts a new one (Self-healing).
  • Services: The "Static IP" or Load Balancer for your pods. Since pods are born and die frequently, Services give them a stable way to talk to each other.

27. Infrastructure as Code (IaC)

Instead of clicking buttons in a cloud console, you write code (like Terraform) to build your whole data center. It's predictable, repeatable, and fast.

Observability Mastery

Don't just run systems; understand them in real-time.

28. The 3 Pillars of Observability

To know if your system is healthy, you need three things:

  • Logs: Discrete events (e.g., "User logged in", "Database connection failed"). Great for debugging what happened.
  • Metrics: Aggregated numbers (e.g., "CPU is at 80%", "99% of requests take < 200ms"). Great for when something is wrong.
  • Traces: The journey of a single request across multiple services. Great for finding where the bottleneck is.

29. The "Gold Standard" Tools

Prometheus & Grafana: The ultimate duo for metrics. Prometheus collects the numbers, and Grafana turns them into beautiful, glowing dashboards.

ELK Stack: Elasticsearch (Search engine), Logstash (Processor), and Kibana (Visualizer). The king of log management.

Advanced Linux Internals

The secret magic that makes modern technology possible.

30. How Docker Actually Works: Namespaces & Cgroups

Docker isn't magic; it uses two core Linux features to create "isolation":

Namespaces (The Illusion)

This gives the process its own private view of the system. It thinks it has its own separate IP, its own users, and its own files, even though it's sharing with everyone else.

Cgroups (The Limits)

Control Groups ensure a single container can't "hog" all the resources. They set hard limits on how much CPU or RAM a process can use.

Performance Tuning & Hardening

Squeezing every drop of power out of your hardware while keeping it safe.

31. Tuning with sysctl

You can change how the Kernel behaves on the fly by editing /etc/sysctl.conf. Want faster networking? Increase the TCP buffer size.

$ sudo sysctl -w net.ipv4.ip_forward=1

32. Security Hardening: SELinux & AppArmor

These are "Mandatory Access Control" systems. Even if a hacker gets root access, they still can't touch files unless the policy explicitly allows it.

CIS Benchmarks: The "Bible" of security. It's a massive checklist of settings to make your server bulletproof.
